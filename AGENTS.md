# WolfChat 协作约束

本文件用于固定后续协作规则，避免重复沟通。

## 1. 代码原则

- 只实现当前确认要用的能力，不做预留兼容分支。
- 禁止冗余代码、死代码、无用依赖、无用样式。
- 保持实现直接、可读、可维护，不为了“可能以后用到”提前复杂化。

## 2. 禁止兜底与兼容

- 禁止防御式兜底默认值来掩盖配置问题（应显式配置，缺失就启动失败）。
- 禁止旧数据/旧字段/旧接口兼容分支。
- 禁止“迁移期双写/双读”这类过渡逻辑长期留在主干。

## 3. 数据库脚本规则

- `database/init.sql` 只用于全新初始化。
- `init.sql` 只保留建库对象定义（表、索引、约束、基础种子数据）。
- 禁止在 `init.sql` 写旧库兼容逻辑（如 `ALTER` 补丁、`INFORMATION_SCHEMA` 判断、动态 SQL 修补）。
- 若未来确需迁移，单独新建迁移脚本，不混入 `init.sql`。

## 4. 注释与风格

- 只保留必要注释：解释业务约束、边界条件、非直观实现原因。
- 删除“描述代码表面行为”的冗余注释。
- 命名与结构优先遵循现有项目风格，避免同义重复字段/概念。

## 5. 变更执行要求

- 每次改动同步清理相关旧实现，不留并行旧入口。
- 提交前自查：是否存在兜底、兼容、冗余三类问题；存在则先清理再提交。
- 若必须偏离本约束，先明确说明原因并单独确认。
