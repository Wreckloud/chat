# WolfChat 产品边界与迭代路线（V1）

## 1. 当前定位

WolfChat V1 定位为：**单聊 + 公共聊天室（Lobby）+ 轻社区论坛**。

- 聊天：互关后私聊（单聊）。
- 聊天室：全站唯一 Lobby，登录用户可读写。
- 社区：任何用户可发主题和回帖，强调自由讨论。
- 治理：仅作者治理，不引入版主体系。

## 2. 群聊边界（本版本结论）

当前版本**不做完整群聊系统**，仅保留单聊与 Lobby。

不做的范围：
- 群实体（建群/加群/退群/解散）
- 群角色权限（群主/管理员）
- 群设置（公告、禁言、邀请）
- 群维度未读与消息分发模型

这样可以把研发精力集中在可稳定交付的核心链路。

## 3. 当前功能范围

### 3.1 聊天

- 单聊：文本 + 图片 + 视频 + 文件 + 链接
- Lobby：文本 + 图片 + 视频 + 文件 + 链接
- 在线状态：在线/上次在线展示与实时推送

### 3.2 社区

- 版块/主题/回复
- 作者治理：锁帖、删帖、删回复、置顶、精华
- 主题列表与回复列表分页

### 3.3 账号

- 狼藉号/邮箱 + 密码登录
- 邮箱认证链接
- 忘记密码（邮箱重置）
- 登录审计记录

## 4. 近期优化顺序（建议）

### Phase A（当前）
- 聊天与 Lobby 前端代码去重（共享发送/ACK/键盘/分组逻辑）
- 消息规则统一（私聊与 Lobby 规则合并）
- 在线状态查询结构优化

### Phase B（随后）
- 社区的话题供给能力：建议话题、每周话题、轻标签
- 社区发布与浏览体验优化

### Phase C（最后）
- 稳定性收口：压测、回归测试、异常链路补齐
- 文档与部署脚本统一

## 5. V1 验收标准

- 新用户可在 5 分钟内完成：登录、发主题、回帖、单聊、进入 Lobby。
- 核心链路无阻断错误：登录、发帖、回帖、发消息、发媒体。
- 页面风格统一，关键交互一致。
